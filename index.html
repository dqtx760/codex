何为Codex

1.1 溯源

Codex原是OpenAI于2021年所创之代码生成模型，曾为GitHub Copilot之根基。如今已蜕变为一套完整的智能体工作流系统。

演变历程：

  • 初代（2021-2023）：专注于代码片段生成
  • 现代（2025至今）：多代理协作平台，可执行复杂任务

1.2 核心：多代理并行

你可同时遣派多名AI弟子：一人重构模块，一人修复破绽，一人演练测试。各弟子独立工作，互不干扰。

1.3 对比

特性传统工具Codex
主要功能代码补全、语法检查任务执行、多代理协作
工作方式单次交互，实时响应多步骤任务，异步执行
上下文理解当前文件或少量文件整个代码库
💡
修炼心得：传统工具更像"代码助手"，而Codex则如"AI队友"——它能理解你的整个项目，独立完成复杂任务。

运行心法

4.1 计划阶段

  1. 明确目标：要解决什么问题？
  2. 分析现状：当前代码结构如何？
  3. 设计方案：如何修改？需要哪些文件？
  4. 定义约束：技术限制是什么？
  5. 制定验证：如何确认完成？

4.2 四字真言

目标：明确说明要构建或修复什么

上下文：提供相关背景信息

约束：定义技术限制

验证：明确"完成"的标准

走火入魔

12.1 常见心魔

Windows用户修炼Codex时，常会遇到以下心魔困扰：

  • 启动报错 0xC0000022 — 沙盒权限问题
  • 文件系统错误 -2143309801 — 应用商店组件异常
  • 每次对话都 Reconnecting — 代理配置问题
  • 应用启动崩溃 — 系统环境冲突

12.2 一般心法

遇到问题时，可先尝试以下通用心法：

  1. 检查Windows版本（需Win10 1903+或Win11）
  2. 确保微软商店能正常打开
  3. 以管理员身份运行
  4. 检查代理/VPN设置

若通用心法无效，说明问题较深，需要针对性诊断。

💡
修炼提示：Windows环境复杂，每个用户的系统配置、代理软件、安全策略都不同。通用方案未必适用于所有人。

神兵利器

13.1 CodexPlusPlus 简介

CodexPlusPlus 是一个 Codex App 的增强工具（17.8k Stars），通过外部注入方式增强 Codex 功能，无需修改原始安装文件。

核心功能：

  • 中转注入：支持接入第三方API，可用国产模型替代
  • 插件解锁：解锁原生被限制的插件入口
  • 会话管理：支持删除、导出、移动会话
  • 用户脚本：支持注入自定义增强脚本

13.2 安装方法

从 GitHub Releases 下载对应系统的安装包：

  • Windows：下载 CodexPlusPlus-*-windows-x64-setup.exe
  • macOS：根据芯片选择 x64 或 arm64 版本

安装后会有两个入口：

  • Codex++：静默启动，直接增强 Codex
  • Codex++ 管理工具：配置面板，管理中转和增强功能

13.3 接入国产模型

通过中转注入功能，可以接入国产模型API，绕过官方限制。

配置示例
# 在 Codex++ 管理工具中配置： 1. 打开"中转注入"页面 2. 确认已检测到 ChatGPT 登录状态 3. 添加中转配置： - Base URL: https://your-api.com/v1 - API Key: sk-xxx 4. 选择配置并应用 5. 启动 Codex++

支持的中转服务商（官方合作）：

  • JOJO Code - 稳定接入，支持GPT-5.5/Claude等
  • AIGoCode - 国内直连，无需魔法
  • APIKEY.FUN - 价格低至官方7%
  • 更多服务商见项目README
💡
注意：中转注入需要官方账号登录态 + 第三方API Key。中转只接管模型请求，账号功能仍由官方服务处理。

降龙十八掌

14.1 Computer Use 是什么

Computer Use 是 Codex 的高级能力，让AI可以直接操控你的电脑屏幕——点击按钮、输入文字、截图识别、操作浏览器，就像一个真人在帮你操作电脑。

核心能力：

  • 屏幕识别：理解屏幕上的UI元素
  • 鼠标操控：点击、拖拽、滚动
  • 键盘输入：打字、快捷键
  • 浏览器操作：网页导航、表单填写

14.2 适用场景

场景示例价值
重复性操作批量填写表格、数据录入解放双手，效率翻倍
软件测试UI自动化测试、回归测试减少人工测试成本
网页操作信息采集、表单提交自动化网页流程
跨应用协作在多个软件间传递数据打通信息孤岛

14.3 实战案例

案例一：自动填写报销单

场景：每周需要从Excel读取数据，手动填入OA系统报销单

Computer Use方案：AI自动打开Excel → 读取数据 → 切换到OA → 逐项填写 → 提交

案例二：网页信息监控

场景：需要每天检查竞品网站的价格变动

Computer Use方案：AI定时打开网页 → 截图识别价格 → 记录到表格 → 发送通知

案例三：批量图片处理

场景：需要给100张产品图添加水印并调整尺寸

Computer Use方案：AI打开图片编辑器 → 逐张处理 → 保存导出

14.4 使用建议

  1. 从简单任务开始：先尝试单一应用内的操作
  2. 明确指令：告诉AI每一步该做什么
  3. 监督执行：初期需要观察AI的操作是否正确
  4. 记录流程：把成功的操作流程保存为模板
